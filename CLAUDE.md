## Project structure

This is an R package.

- R/ — Source files
- tests/testthat/ — Tests (convention: R/foo.R -> tests/testthat/test-foo.R)
- man/ — Generated docs (do not edit)
- NAMESPACE — Generated by roxygen2 (do not edit)
- DESCRIPTION — Package metadata and dependencies

## Development commands

Run these in an R session (or via Makefile targets):

    devtools::load_all()     # load package for dev
    devtools::test()         # run all tests
    devtools::check()        # full R CMD check
    devtools::document()     # regenerate roxygen docs

Run this in the shell:

    air format .             # format all R files

## Code style

Read R/[exemplar].R and its test file to understand the style.

- Use pkg::fun() for external package calls (never bare library() in R/ code)
- Use rlang::abort() / warn() / inform() instead of stop() / warning() / message()
- Do NOT add code comments except roxygen documentation
- Do NOT remove existing comments unless the related code is also removed
- Place exported functions at top of files, internal helpers below

## Testing rules

- Use expect_snapshot(error = TRUE) for testing error conditions
- NEVER delete or mock away failing tests — fix the source code
- NEVER add complex mocking to make tests pass

## Constraints

- MUST run devtools::test() before committing
- MUST NOT modify files in man/ directly
- MUST NOT edit NAMESPACE directly — use roxygen2 tags
- MUST run devtools::document() after changing any roxygen comments
